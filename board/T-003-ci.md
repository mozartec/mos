# T-003 — CI

GitHub Actions running build, lint, and Vitest on every push and PR, on Bun.

## Outcome

After this task, every push and pull request runs build, lint, and test across the workspace
on CI (GitHub Actions, Bun), with Turbo orchestrating and caching. A red run blocks merge; a
status badge on `main` reflects health. This is the cheap guardrail that keeps the
cold-agent workflow honest — broken cards can't silently land.

## Context — read before starting

- [`docs/04-TECH_STACK.md`](../docs/04-TECH_STACK.md) — Bun + Turbo + Vitest + ESLint/
  Prettier are the commands CI runs (`bun run build|lint|test`).
- [`docs/11-RELEASING.md`](../docs/11-RELEASING.md) — Conventional Commits, squash-merge,
  SemVer; CI is the gate before merge.
- [`.agents/skills/turborepo/`](../.agents/skills/turborepo) — for Turbo remote/local
  caching in CI and `--filter`/affected-only runs.
- [`scripts/validate-vault.mjs`](../scripts/validate-vault.mjs) — also run `bun run validate`
  in CI so the board (this repo as a vault) stays valid.
- [`.github/`](../.github) — existing workflows/config to extend, not duplicate.

## Constraints (must honor)

- Run via the root Turbo scripts so CI matches local (`bun run build|lint|test`); don't
  hand-roll per-package commands. (ADR-008)
- Keep it Bun-based (no npm/yarn in CI) to match the workspace.
- Fast feedback: cache Bun installs and Turbo outputs; fail the job on any task failure.

## Plan

1. A GitHub Actions workflow on push + PR: checkout, setup Bun, `bun install`, then
   `bun run lint`, `bun run test`, `bun run build`, and `bun run validate`.
2. Enable caching (Bun store + `.turbo`). Make the job required for merge.
3. Add a status badge to `README.md`.

## Acceptance

- [x] A red build blocks merge; the badge is green on main.
- [x] CI runs lint + test + build + vault-validate via the root Turbo scripts on Bun.
- [x] Caching is enabled so unchanged packages aren't rebuilt/retested.
